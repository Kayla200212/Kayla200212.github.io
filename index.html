First Dataset: Grocery Stores in Chicago 2013

Our first data set is one that provides a list of all grocery stores in Chicago from 2013. This dataset comes from the City of Chicagos business license data. Since this dataset is slightly outdated (one grocery store that appears in the dataset is Dominicks, a store that went out of business), we would like to see if there is a way for us to filter or clean the business license data to get the most up to date list of grocery stores. We also noticed that some of the grocery stores included in the list are convenience stores such as Walgreens. While Walgreens does technically have groceries, its not a fully stocked grocery store so we decided to filter it out of the dataset. We would like to take a closer look at the stores included in this dataset in particular, as there might be other stores similar to Walgreens we would like to exclude. Choices such as these can help further guide our ETL steps as we gather and combine more datasets. The two most important variables in this data set are the name of the grocery store and the neighborhood the grocery store is in. One question we can ask using these metrics: which neighborhoods have the least amount of grocery stores? By using this as a starting point, we can use this to begin analysis of food inequality through comparison with other datasets.

Initial Findings & Future Work

From our initial visualizations, the connection is vague. We visualized how the number of grocery stores per community area related to the hardship index and percentage of households below poverty, respectively, and both had slight positive correlations which would indicate that an increase in stores meant an increase in poverty indicators. This was not what we were expecting, but two things could improve the certainty of this result.

For one, as above, creating a more recent and more full dataset of stores would help greatly. At present, most of the communities have less than 6 stores in them, with a few skewing towards having more than 15. More data on grocery stores in these areas would create more validity to our findings. Secondly, if we are able to classify the price range of each grocery store, this would also help us understand these results on a deeper level. Even if an area has a large amount of stores, they could be dollar stores or stores comparable to Walgreens. Knowing the price range of the stores included in the survey would also allow us to better understand the data and results.

Examples
--------

    The simplest way to use nbconvert is

            > jupyter nbconvert mynotebook.ipynb --to html

            Options include ['asciidoc', 'custom', 'html', 'latex', 'markdown', 'notebook', 'pdf', 'python', 'qtpdf', 'qtpng', 'rst', 'script', 'slides', 'webpdf'].

            > jupyter nbconvert --to latex mynotebook.ipynb

            Both HTML and LaTeX support multiple output templates. LaTeX includes
            'base', 'article' and 'report'.  HTML includes 'basic', 'lab' and
            'classic'. You can specify the flavor of the format used.

            > jupyter nbconvert --to html --template lab mynotebook.ipynb

            You can also pipe the output to stdout, rather than a file

            > jupyter nbconvert mynotebook.ipynb --stdout

            PDF is generated via latex

            > jupyter nbconvert mynotebook.ipynb --to pdf

            You can get (and serve) a Reveal.js-powered slideshow

            > jupyter nbconvert myslides.ipynb --to slides --post serve

            Multiple notebooks can be given at the command line in a couple of
            different ways:

            > jupyter nbconvert notebook*.ipynb
            > jupyter nbconvert notebook1.ipynb notebook2.ipynb

            or you can specify the notebooks list in a config file, containing::

                c.NbConvertApp.notebooks = ["my_notebook.ipynb"]

            > jupyter nbconvert --config mycfg.py

To see all available configurables, use `--help-all`.
